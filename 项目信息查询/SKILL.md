# 项目信息查询 Skill

## 任务

输入：产品名/项目名/公司名/一篇文章/一个链接（任一形态）。

输出：**严格按下方 5 段编号格式**直接在对话里返回。不写文件、不写表格。

## 三条铁律

1. **不编造**：搜不到的不写。**邮箱不能从域名+姓名格式拼凑**（`firstname.lastname@company.com` 这种推断永远不输出）。
2. **来源驱动**：每条信息都要在内部有可追溯的来源链接（用于自我判断置信度），但来源链接**不在最终输出中展示**。
3. **置信度门槛**：内部判断为【低】置信度的内容不输出（如仅凭同名匹配但无前公司/学校交叉验证的人选、HR/招聘通用邮箱）。

## 搜索预算与停止条件（防止陷入循环）

项目信息查询本质上必有缺失，**80% 的有用信息能用 20% 的时间拿到，再深挖性价比骤降**。超过预算时，按已搜到的信息直接输出 5 段，缺失字段就缺失——不要把"还没搜全"当作不能输出的理由。

- **总查询预算**：单个项目最多 **12 次** `web_search` 调用。**第 8 次起开始评估"是否已收敛"**，到 12 次必须停下输出
- **单实体上限**：同一个人名/公司名/产品名最多搜 **3 个不同维度** 的关键词（如"姓名+公司"、"姓名+前公司"、"姓名+学校"）。3 次都没显著新信息就放弃这个实体——**禁止只换语序或加同义修饰词反复刷**（`"创始人 清华"`、`"清华 创始人"`、`"创始人 清华大学"` 算同一次）
- **web_fetch 失败即放弃该 URL**：报 404 / enterprise policy block / 超时 → 立刻跳过，**不要换协议、换子路径、换备用域名反复试**。改用 `web_search` 拿替代源，仍失败就放弃这条线
- **并行执行**：同一批可独立的查询**一次性发出**（一条消息内 4-6 个 `web_search` 调用），**禁止串行一条一条等**
- **方向收敛判定**（任一达成即停止该方向，转去下一个）：
  - **团队**：找到 1-2 位核心成员的姓名 + LinkedIn 或多段过往经历
  - **融资**：找到最近一轮的轮次/领投/时间（多轮更佳，单轮也算收敛）
  - **公司联系方式**：4 项必查（官网/LinkedIn/Twitter/邮箱）至少找到 2 项
- **没人就是没人**：搜了一圈所有 创始人 / CEO / founder 关键词都没人名出来，说明这个项目人员信息**就是没公开**——直接在第 4 段写"暂无公开团队信息"，第 5 段只列公司渠道。**不要换 5 个英文别名再搜 5 轮**

## 核心搜索循环

**不按顺序执行。按"我现在手上有什么"决定下一步搜什么。** 每发现一个新实体（人名/公司名/域名/社交账号），立刻把它作为输入触发对应分支，循环扩展直到信息收敛。

中文输入主走中文渠道，英文输入主走英文渠道，不确定就两边都搜。LinkedIn / X / GitHub 不分国内外都要试。

### 拿到「公司名」 → 做这些

```
"{公司名}" 融资                                      ← 中文项目最高效，融资报道里通常包含创始人+背景+投资方
"{公司名}" 36kr | 虎嗅 | 创业邦 | 量子位 | IT桔子      ← 中文媒体专题报道
"{公司名}" 创始人 / "{公司名}" 法人                    ← 直接搜创始人
"{公司英文名}" funding | raised | Series A|B|C        ← 英文项目按轮次扫融资
site:techcrunch.com "{公司英文名}"                   ← TechCrunch 融资专栏
site:crunchbase.com "{公司英文名}"                   ← Crunchbase（含 founder、location、funding rounds）
site:linkedin.com/company "{公司英文名}"             ← 公司主页 + 员工列表
site:linkedin.com/in "{公司英文名}"                  ← 直接搜该公司员工，排名靠前的通常是创始人
```

→ web_fetch 公司官网，重点看 `/about` `/team` 和**页脚**（`mailto:` 邮箱、社交链接、版权声明里的公司全名）。

→ 搜出来的人名、域名、社交链接，**全部作为新实体继续循环**。

**公司层面联系方式必查（不能漏）**：每个项目都要主动检索以下四项，不能只顾找成员忽略公司：

```
公司官网：从最初输入或公司搜索结果里直接拿到
公司 LinkedIn：site:linkedin.com/company "{公司英文名}"   ← 直接锁定公司主页
公司 Twitter/X：site:x.com "{公司英文名}" / "{公司英文名}" twitter / 官网页脚 social icon
公司邮箱：web_fetch 官网 /contact /about + 页脚 mailto: + 「{公司名} contact email」搜
```

公司主页常见路径：`linkedin.com/company/{slug}`、`x.com/{handle}`、`{domain}/contact`。**这些找到了就写到第 5 项的"公司"前缀行里**。

如果搜过程中还冒出 YouTube / Discord / 公司公众号 / GitHub Org / Bilibili 等其他公司层面渠道，作为公司块的追加行写在必查四项之后。

### 拿到「人名」 → 做这些

```
"{英文名/拼音}" linkedin                              ← 海外项目最高价值
"{英文名}" "{前公司}" linkedin                        ← 知道前公司，精准度极高（防同名）
"{英文名}" "{学校}" linkedin                          ← 学校也能锁定
"{英文名}" twitter / "{英文名}" "@" site:x.com       ← X 账号
"{英文名}" github                                    ← 技术创始人大概率有

"{中文名}" "{公司名}" / "{中文名}" "{前公司}"           ← 国内项目主打用前公司锁定身份
"{中文名}" 专访 / "{中文名}" 演讲 / "{中文名}" 采访     ← 中文媒体专访常含完整人物背景段落，比 LinkedIn 还详细
```

**找到 profile 后必须 web_fetch 取实质内容**，不能只记链接。提取：教育背景、完整职业轨迹、创业经历、技术成就、页面里出现的**其他联系方式**（邮箱、个人站点、其他社交链接）。

→ 页面里出现的**新前公司、新链接**作为新实体继续循环。

### 拿到「域名/URL」 → 做这些

- web_fetch 主页 + `/about` `/team` `/contact`，提取页脚 `mailto:` 邮箱、社交链接、公司全名

### 拿到「社交账号/handle」 → 做这些

- LinkedIn handle → web_fetch 个人页拿完整履历
- X handle → web_fetch 主页（bio + pinned tweets，常有邮箱/网站）
- GitHub username → `https://api.github.com/users/{username}` 拿结构化 JSON（含 name、company、blog、email、bio、location）
- 同一 handle 名常跨平台一致 → 拿到后**用相同字符串去其他平台搜一遍**

## 两个反复出现的陷阱

1. **同名问题（中文项目高发）**：搜"王伟、张磊、李明"等常见名时大量同名。**绝对不要凭直觉排除**——投资人/大厂高管/学者转创业者在国内极常见。每个同名候选至少花一步搜 `"{候选人名}" "{候选人所在机构}" "{目标公司名}"` 验证。验证不通过的不要输出。

2. **团队背景描述 ≠ 创始人个人背景**：媒体或招聘文案里写「核心团队来自字节、Google、清华」描述的是团队**集体**背景，**不代表 CEO 本人也来自这些公司**。CEO 可能完全是另一条路径（投资人转创业、学者创业、跨行业创业）。**不能用团队背景描述去搜 CEO 个人 LinkedIn**——必须通过"公司名+融资/创始人"独立确认 CEO 真实履历。

## 融资信息深挖

- **完整融资史，不止最新一轮** —— 种子→A→B 全部按时间顺序串联进同一段
- **领投/跟投分清** —— 报道里写「红杉领投，XX 跟投」按层级写
- **金额单位标清** —— 人民币 vs 美元、万 vs 千万 vs 亿
- **时间精确到月或具体年份** —— 融资报道一般有具体月份
- **未披露照实写** —— 写「金额未披露」

## 输出格式（严格按此结构）

直接在对话里返回，**严格按以下 5 段编号格式**。不要发挥、不要加 emoji、不要加额外章节。

```
1. 项目名称：{中文名}（{英文名}）｜ 官网：{URL}

2. 项目介绍：
{一段自然语言，2-3 句，做什么、目标用户、核心亮点/技术方向}

3. 融资情况：
{一段自然语言，按时间顺序把每一轮融资串联起来。例：2025 年初天使轮近千万美元红杉中国领投，2025 年 11 月 Pre-A 轮数千万美元腾讯投资领投}

4. 团队介绍：

- {姓名}（{职位}）：{教育背景 + 完整职业轨迹 + 创业经历 + 技术成就，每位 80-150 字}

- {姓名2}（{职位2}）：{同上密度}

5. 联系方式：

- {成员姓名} LinkedIn: {URL}
- {成员姓名} Twitter: {URL}
- {成员姓名} 邮箱: {email}

- {成员姓名2} LinkedIn: {URL}

- 公司官网: {URL}
- 公司 LinkedIn: {URL}
- 公司 Twitter: {URL}
- 公司邮箱: {email}
```

### 第 4 项团队介绍密度要求

每位成员的背景**必须达到 80-150 字**密度，不能只用一句"前 XX 公司 XX 职位"敷衍。每位成员必须包含：

- **教育背景**：学校 + 专业，有学位/年份更好
- **完整职业轨迹**：按时间顺序，每段经历写清「时间段 + 公司 + 职位 + 核心职责或成就」。多段经历都要列，不要只挑一段
- **创业经历**（如有）：之前办过的公司、退出/收购情况
- **技术成就**（如有且适用）：论文、开源项目、专利、获奖

**禁止只写当前职位**——"CEO 负责战略与生态"、"CTO 主导核心研发" 这种没有过往经历支撑的描述对投资人没有判断价值。

**搜不到完整信息时**：照实写已确认的部分，不编造。比如只搜到一段经历就写一段，**不要用"曾任 XX" 凑数**，**不要编年份和学校**。

### 参考样例（按现行规则的完整示范）

```
1. 项目名称：言创万物（Verdent）｜ 官网：https://verdent.ai

2. 项目介绍：
言创万物（Verdent）是一家 2025 年初成立的 AI Coding 公司，旗舰产品 Verdent 是定位「AI 时代资深开发助手」的 VS Code 插件，主打 Subagent 多智能体并发任务执行与自动代码质量验证。

3. 融资情况：
2025 年初天使轮近千万美元红杉中国领投，2025 年 11 月 Pre-A 轮数千万美元腾讯投资领投。

4. 团队介绍：

- 陈志杰（CEO）：前字节跳动技术高级总监、TikTok 推荐算法负责人，主导海外推荐系统架构升级。早年在百度任首席技术架构师，长期深耕推荐与广告系统。2025 年初创办 Verdent，主导整体技术与产品方向。

- 刘晓春（联合创始人）：11 年百度凤巢算法团队工作经验，参与凤巢多代排序模型迭代。2025 年与陈志杰联合创办 Verdent。

5. 联系方式：

- 陈志杰 LinkedIn: https://sg.linkedin.com/in/zhijie-chen-bytechen

- 公司官网: https://verdent.ai
- 公司 LinkedIn: https://www.linkedin.com/company/verdent
```

注意样例里几个关键点：
- 第 1 段官网用全角 `｜` 分隔到同一行
- 第 4 段每位成员**括号注职位**，背景陈述含多段经历（即使信息有限也要尽量写时间/机构）
- 第 5 段同一主体的多条联系方式**连续列（组内紧凑、bullet 之间无空行）**，不同主体之间**空一行分隔**
- 第 5 段**没有尾随分号或多余空格**，URL / email 后直接换行
- 第 5 段没找到的渠道（陈志杰 Twitter / 邮箱、刘晓春 LinkedIn / Twitter / 邮箱、公司 Twitter / 邮箱）**整行省略**

### 格式硬规则

- **严格用 `1.` `2.` `3.` `4.` `5.` 五段编号**，不用 `【】`、emoji、其他章节标题
- **第 1 项格式**：`1. 项目名称：{中文名}（{英文名}）｜ 官网：{URL}` 一行写完，使用全角 `｜` 分隔。如果只有英文名或只有中文名，括号内省略；如果搜遍后仍找不到官网，`｜ 官网：` 那段省略
- **第 2 项「项目介绍」、第 3 项「融资情况」必须是自然语言段落**，不分点、不分行、不带链接
- **第 4 项「团队介绍」格式**：`- {姓名}（{职位}）：{背景}`——括号注职位，背景每位 80-150 字，含教育 + 完整职业轨迹 + 创业经历 + 技术成就（详见上方密度要求）
- **第 5 项「联系方式」格式**：`- {主体} {渠道}: {值}`——半角冒号 + 单空格 + 值，**值后直接换行，不要尾随分号或多余空格**
- **未找到的联系方式整行省略**：找到才出现，没找到不要保留字段名留空。某成员只找到 LinkedIn 就只列 LinkedIn 一行；某成员一个联系方式都没找到，就只在第 4 项写背景，第 5 项不列其名下任何条目。可查找的渠道范围：LinkedIn / Twitter（X）/ 邮箱 / GitHub / 微信公众号 / 脉脉 / ProductHunt / 个人站点。**不查找即刻和小红书**
- **公司层面四项必查必列**：每次执行都要主动检索 **公司官网**、**公司 LinkedIn**、**公司 Twitter/X**、**公司邮箱**——找到的写进第 5 项联系方式，统一用「公司」前缀。如果搜遍了官网/页脚/LinkedIn 公司页/X 等渠道仍未找到，那一行才省略
- **第 5 项内部分组排序**：
  - **团队成员组在前**（按第 4 项里出现顺序分组排列）
  - **公司组在最末**
  - **组内紧凑**：同一主体的多条联系方式连续放置，bullet 之间**无空行**
  - **组间空行**：不同主体（不同成员之间、最后一位成员与公司组之间）**空一行**
- **联系方式字段只放纯净的 URL / email / handle**，**严禁混入地址、备案号、注册资本、电话、办公室地点等无关信息**。如果搜索结果把多种信息黏在一起，输出前必须截取出干净的 URL 或邮箱，去掉后缀
- **段落行距**：第 1～5 段之间空一行；第 4 项各成员 bullet 之间空一行
- **不展示置信度标签**（【高/中/低】不出现在最终输出里，仅作内部判断依据）
- **不展示来源链接**（`← 来源` 不出现在最终输出里，仅作内部判断依据）
- **不写表态**（不写"建议联系 XX"）
- **不写过程废话和注释**（不写"经过多轮搜索"、"由于公开信息有限"、"注：xxx"、"公开信息暂未披露"）

## 自主探索

上面是保底路径。走完还不够时不要停——你有 web_search 和 web_fetch，能搜任何公开信息。换关键词组合（名字+公司、名字+前公司、名字+学校结果可能完全不同）、换平台、换语言。**每条新信息都是新搜索输入**。

## 边界

- 不写飞书/Notion/任何外部表
- 不生成本地文件、不写报告
- 所有结果只在当前对话里返回
- 不做深度业务分析——本 Skill 只交付**项目介绍 + 融资 + 团队 + 联系方式** 5 段
